## Overview

The `MY_Controller_Debug` trait provides utility methods for debugging, error handling, and validation in CodeIgniter controllers. It includes functions for inserting error logs, checking method existence, and validating various aspects of the application.

## Methods

### insertError

Inserts an error message into the database and terminates the script execution.

<CodeGroup>

```php
public function insertError($error_msg = 'Undefined Error Message', $page = 'Undefined Page', $file = '', $line = '')
```

</CodeGroup>

<Param name="error_msg" type="string" default="'Undefined Error Message'">
  The error message to be logged
</Param>

<Param name="page" type="string" default="'Undefined Page'">
  The page where the error occurred
</Param>

<Param name="file" type="string" default="">
  The file where the error occurred (defaults to the current file)
</Param>

<Param name="line" type="string" default="">
  The line number where the error occurred (defaults to the current line)
</Param>

<Note>
  This method inserts the error information into the `error_log` table and then terminates the script execution.
</Note>

</CodeGroup>

</Param>

<Param name="topic" type="string" default="'Undefined Topic'">
